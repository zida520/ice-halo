网页程序通过 HTTP 上传文件，Flask 后端接收后调用封装好的模型函数进行推理，最后将结果以 JSON 格式返回给前端显示

app.py文件是后端吗，后端在虚拟环境中运行，调用网页和调用模型程序和返回结果

当我们在虚拟环境中运行app.py返回一个数字网址，
这个数字网址是 Flask 后端服务监听的地址，并不是“前端”本身，但你的前端页面（index.html）是通过这个地址来访问的

你访问 http://127.0.0.1:5000 时，实际上是向 Flask 后端发送了一个 HTTP 请求，后端收到请求后，把 index.html 文件的内容作为响应返回给浏览器

发生了什么网络请求？
浏览器向 http://127.0.0.1:5000/upload 这个地址（即后端地址）发送一个 POST 请求。

请求头中自动添加 Content-Type: multipart/form-data; boundary=...，表示请求体中有文件。

请求体包含音频文件的二进制数据以及相关字段。

后端做了什么
解析 HTTP 请求，取出上传的文件。

调用模型函数，得到结果字典（如 {"transcription": "hello", "processing_time_seconds": 1.23}）。

jsonify() 将字典转换成 JSON 格式的字符串（如 '{"transcription": "hello", "processing_time_seconds": 1.23}'）。

把这个 JSON 字符串作为 HTTP 响应的 body，加上 Content-Type: application/json 头部，返回给客户端。
前端接收响应并更新页面

response.json() 把后端返回的 JSON 字符串变成 JS 对象。

通过 data.transcription 取出值，并更新 HTML 元素。

前端和后端不共享任何变量或内存，只能通过这种请求-响应模式交换信息。这也是为什么你需要使用 jsonify 和 response.json() 来“打包”和“拆包”数据
