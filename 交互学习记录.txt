激活虚拟环境，后运行app.py，然后打开浏览器输入网址，然后成功运行，说说原理


浏览器发送 POST /upload (multipart/form-data)
Flask 接收文件 → 保存到临时文件 → 获得路径 tmp_path
调用 process_data(tmp_path)
返回 {"transcription": "文本"}
Flask 用 jsonify 转为 HTTP 响应
浏览器接收 JSON，解析并显示结果

网页程序通过 HTTP 上传文件，Flask 后端接收后调用封装好的模型函数进行推理，最后将结果以 JSON 格式返回给前端显示

app.py文件是后端吗，后端在虚拟环境中运行，调用网页和调用模型程序和返回结果

当我们在虚拟环境中运行app.py返回一个数字网址，
这个数字网址是 Flask 后端服务监听的地址，并不是“前端”本身，但你的前端页面（index.html）是通过这个地址来访问的

你访问 http://127.0.0.1:5000 时，实际上是向 Flask 后端发送了一个 HTTP 请求，后端收到请求后，把 index.html 文件的内容作为响应返回给浏览器
